# ピクロス自動解答ビューアー 仕様書

## 概要

ピクロス（ノノグラム）パズルをアルゴリズムが自動で解いていく様子を、ユーザーがリアルタイムで眺めるためのアプリケーションです。操作は最小限とし、「見て楽しむ」体験に特化します。

---

## 機能要件

### 1. パズル管理

- **プリセットパズル**: 5×5 / 10×10 / 15×15 / 20×20 のサイズ別にパズルを複数内蔵
- **ランダム生成**: 指定サイズのランダムなピクロスを生成（解が一意になるよう検証）
- パズルの切り替えは「次のパズル」ボタン1つのみ

### 2. 自動解答エンジン

#### アルゴリズム構成（段階的に適用）

| フェーズ | 手法                     | 説明                                       |
| -------- | ------------------------ | ------------------------------------------ |
| 1        | ライン解析（確定）       | 各行・列のヒント数から必ず埋まるセルを確定 |
| 2        | ライン解析（消去）       | 必ず空白になるセルを確定                   |
| 3        | 交差解析                 | 行と列の確定情報を突き合わせて推論         |
| 4        | 仮定法（バックトラック） | 上記で解けない場合、1セルを仮定して探索    |

#### ステップ管理

- 解答過程を「ステップ」単位で記録（1ステップ＝1セルの確定）
- 各ステップにメタデータを付与：
  - 対象セル座標 (row, col)
  - 確定種別（塗り / 空白）
  - 適用したアルゴリズムフェーズ
  - 根拠となった行番号または列番号

### 3. 再生・表示

#### 再生コントロール

| 操作           | 動作                                |
| -------------- | ----------------------------------- |
| ▶ 再生         | 自動ステップ送り開始                |
| ⏸ 一時停止     | 再生を止める                        |
| ⏭ スキップ    | 即座に完成状態を表示                |
| 速度スライダー | 1ステップあたりの間隔：50ms〜2000ms |

#### ビジュアル演出

- セル確定時にアニメーション（フェードイン・フラッシュ等）
- 現在処理中の行・列をハイライト
- フェーズごとにセルの色を変えて視覚的に区別：
  - フェーズ1（確定塗り）：メインカラー
  - フェーズ2（確定空白）：グレー系
  - フェーズ4（バックトラック）：アクセントカラー
- 完成時に完了アニメーション（全セルが波打つ等）

### 4. 情報パネル

- 現在のステップ数 / 総ステップ数
- 適用中のアルゴリズムフェーズ名
- 経過時間
- 解答済みセル数 / 総塗りセル数

---

## 非機能要件

### UI・UX

- **操作は眺めるだけ**を前提とし、UIは極力シンプル
- 初回ロード後、自動的に再生を開始する（オプション設定可）
- レスポンシブ対応：デスクトップ・タブレット・モバイルで視認性を確保
- グリッドはウィンドウサイズに合わせてスケール

### パフォーマンス

- 20×20 パズルの全ステップ事前計算を 500ms 以内に完了
- アニメーションは 60fps を維持

### ブラウザ対応

- Chrome / Firefox / Safari / Edge 最新版

## 技術仕様

### 実装形式

- パッケージマネージャ: pnpm
- フレームワーク: SolidJS
- スタイリング: 生CSS
- アプリケーション構成: SPA
- ストレージ: 不要（状態はメモリ内のみ）

### データ構造

```js
// パズル定義
{
  size: 10,
  rowClues: [[3, 2], [1], ...],   // 各行のヒント
  colClues: [[4], [2, 1], ...],   // 各列のヒント
  solution: [[1,0,1,...], ...]    // 正解グリッド（検証用）
}

// ステップ定義
{
  row: 3,
  col: 5,
  value: 1,           // 1=塗り, 0=空白
  phase: 1,           // 1〜4
  sourceAxis: 'row',  // 'row' | 'col'
  sourceIndex: 3      // 根拠となった行/列のインデックス
}
```

### アルゴリズム詳細

**ライン解析（フェーズ1・2）**

各行・列について以下を計算します。

1. ヒント列を左詰めに配置したときの最左位置（`leftmost`）
2. 右詰めに配置したときの最右位置（`rightmost`）
3. `leftmost[i] == rightmost[i]` のセルを「確定塗り」
4. 全配置パターンで必ず空白になるセルを「確定空白」

**バックトラック（フェーズ4）**

未確定セルを1つ選び、塗り／空白を仮定してフェーズ1〜3を再適用します。矛盾が生じた場合は逆の値を確定し、ステップとして記録します。
